TRAIL SCAN V5 — วิธีนำไปใช้งาน
================================

กรณีติดตั้งใหม่
----------------
1. ใส่ Supabase URL และ Publishable/Anon Key ใน assets/config.js
2. รัน SQL ตามลำดับ:
   001_trail_scan_core_schema.sql
   002_trail_scan_security_rls.sql
   003_trail_scan_rpc.sql
   004_verify_trail_scan_installation.sql
   005_user_access_dashboard_public_results.sql
3. อัปโหลดไฟล์เว็บไซต์ทั้งหมดขึ้น Hosting ผ่าน HTTPS
4. สมัครบัญชีแรก แล้วเปิด pending.html
5. กด “ตั้งเป็นผู้ดูแลระบบคนแรก”
6. สร้างองค์กรและเริ่มกำหนดผู้ใช้งาน

กรณีอัปเกรดจาก V4
-------------------
1. สำรองฐานข้อมูลและไฟล์เว็บไซต์เดิม
2. เก็บค่าจาก assets/config.js เดิมไว้
3. อัปโหลดไฟล์ V5 ทับชุดเดิม
4. ใส่ค่า Supabase เดิมกลับใน assets/config.js
5. รันเฉพาะ supabase/005_user_access_dashboard_public_results.sql
6. Logout/Login ใหม่
7. เปิดเมนู “ผู้ใช้งานและสิทธิ์” แล้วตรวจบทบาท Event และจุด Scanner

พฤติกรรมสำคัญ
--------------
- บัญชีใหม่เป็น PENDING และเข้าใช้งานไม่ได้จนกว่า Admin อนุมัติ
- ผู้ใช้เห็นเฉพาะเมนู Event และจุด Scanner ที่ได้รับมอบหมาย
- สิทธิ์ถูกตรวจทั้งหน้าเว็บ Supabase RLS และ RPC
- นักวิ่งตรวจผลเองได้ที่ results.html โดยเลือก Event และกรอก BIB
- Event ต้องเปิด public_results_enabled และเลือกรูปแบบ LIVE หรือ FINAL_ONLY

คำเตือน
-------
ห้ามใส่ service_role key, database password หรือ secret key ใน assets/config.js
ระบบกล้องและ PWA ต้องเปิดผ่าน HTTPS
