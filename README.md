# Trail Scan Web V5 - Access Control & Public Results

เว็บชุดใหม่ทั้งหมดสำหรับระบบสแกนงานวิ่ง โดยไม่ใช้ Vercel, Python, Node.js หรือระบบ Build

## ขอบเขต

- Login / Supabase Auth
- หลายองค์กร
- หลาย Event
- ประเภทการแข่งขัน
- เวลาปล่อยตามกำหนดและเวลาปล่อยจริง
- นักวิ่งและ QR Token
- Start
- CP เพิ่มได้ไม่จำกัด
- Finish
- มอบหมายเจ้าหน้าที่ประจำจุด
- Dashboard
- ผลการแข่งขัน
- Rapid Scanner
- IndexedDB Offline Queue
- Sync ผ่าน RPC `record_scan()` และ `sync_scan_batch()`

## 1. ใส่ค่า Supabase

แก้ไฟล์ `assets/config.js`

```javascript
window.TRAIL_SCAN_CONFIG = Object.freeze({
  APP_NAME: 'Trail Scan',
  SUPABASE_URL: 'https://YOUR_PROJECT_ID.supabase.co',
  SUPABASE_ANON_KEY: 'YOUR_PUBLISHABLE_OR_ANON_KEY',
  DEFAULT_TIMEZONE: 'Asia/Bangkok',
  SCANNER: { FPS: 18, DUPLICATE_LOCK_MS: 4000, RESULT_MS: 1200, BATCH_SIZE: 30 }
});
```

ใช้เฉพาะ Publishable Key หรือ Legacy Anon Key

ห้ามใส่:
- service_role key
- database password
- secret key
- Telegram Bot Token

## 2. รัน SQL ใน Supabase

ไฟล์อยู่ในโฟลเดอร์ `supabase/`

1. `001_trail_scan_core_schema.sql`
2. `002_trail_scan_security_rls.sql`
3. `003_trail_scan_rpc.sql`
4. `004_verify_trail_scan_installation.sql` สำหรับตรวจสอบ

## 3. อัปโหลดขึ้น Hosting

อัปโหลดไฟล์ทั้งหมดไปยัง `public_html` หรือโฟลเดอร์ Subdomain ของคุณ เช่น

`https://trail.yourdomain.com`

ต้องใช้ HTTPS เพื่อให้กล้องและ PWA ทำงาน

## 4. ลำดับใช้งาน

1. เปิด `login.html`
2. สมัครหรือ Login
3. สร้างองค์กร
4. สร้าง Event
5. สร้างประเภทการแข่งขัน
6. สร้าง START, CP และ FINISH
7. เพิ่มหรือนำเข้านักวิ่ง
8. มอบหมายเจ้าหน้าที่
9. เปิด `scanner/index.html`
10. กดดาวน์โหลดข้อมูล Offline ก่อนออกไปประจำ CP

## Offline

ทุกการสแกนถูกเก็บลง IndexedDB ในมือถือก่อน เมื่ออินเทอร์เน็ตกลับมา ระบบ Sync เข้า Supabase อัตโนมัติ โดยใช้ `offline_id` ป้องกันรายการซ้ำ


## เวอร์ชัน V3: เพิ่มระบบลบครบทุกหน้าจัดการ

เพิ่มปุ่มลบใน:
- องค์กร
- Event
- ประเภทการแข่งขัน
- Start / CP / Finish
- นักวิ่ง
- เจ้าหน้าที่
- อุปกรณ์ Scanner

### หลักการป้องกันข้อมูลเสียหาย

- ลบองค์กรหรือ Event ต้องพิมพ์ชื่อยืนยัน และแสดงจำนวนข้อมูลที่จะถูกลบ
- ประเภทที่มีนักวิ่ง/สแกน/ผลแล้ว จะไม่ให้ลบ แต่ปิดใช้งานแทน
- จุดสแกนที่มีประวัติแล้ว จะไม่ให้ลบ แต่ปิดใช้งานแทน
- นักวิ่งที่มีประวัติแล้ว จะไม่ให้ลบ แต่เปลี่ยนเป็น CANCELLED แทน
- เจ้าหน้าที่และอุปกรณ์ที่ไม่ต้องใช้สามารถลบได้
- หน้าผลการแข่งขันไม่มีปุ่มลบ เพราะเป็นข้อมูลที่ระบบคำนวณจาก Scan Logs


## เวอร์ชัน V4 - QR และป้าย BIB

เพิ่มความสามารถ:
- ดาวน์โหลด QR นักวิ่งรายคนเป็น PNG ขนาดสูง
- ดาวน์โหลด QR นักวิ่งทั้งหมดหรือเฉพาะที่เลือกเป็น ZIP
- แยกโฟลเดอร์ QR ตามรหัสประเภทการแข่งขัน
- เมนูจัดทำป้าย BIB พร้อม QR
- เลือกนักวิ่งรายคน ทั้งประเภท หรือทั้งหมด
- กำหนดขนาดป้าย ขนาด QR ตำแหน่ง QR สี โลโก้ และข้อความ
- รองรับชื่อภาษาไทยโดยวาดป้ายลง Canvas ก่อนใส่ PDF
- จัดหลายป้ายต่อกระดาษ A4/A3 อัตโนมัติ
- บันทึกแบบป้ายไว้ใน Browser เครื่องที่ใช้งาน

### ข้อแนะนำการพิมพ์ QR
- QR บนป้ายควรไม่น้อยกว่า 30 x 30 มม.
- ค่าที่แนะนำคือ 35-40 มม.
- ใช้พื้น QR สีขาวและห้ามวางทับรอยพับ/รูเจาะ
- ทดสอบสแกนป้ายจริงก่อนพิมพ์ทั้งงาน


## คู่มือการใช้งาน

อยู่ในโฟลเดอร์ `manual/` ทั้ง PDF และ Word และเปิดได้จากปุ่ม “คู่มือการใช้งาน” ในหน้า Admin


## เวอร์ชัน V5 - อนุมัติผู้ใช้ สิทธิ์ตามหน้าที่ และผลการแข่งขันสาธารณะ

### การสมัครและอนุมัติ
- บัญชีใหม่มีสถานะ `PENDING` และยังใช้งานไม่ได้
- ผู้สมัครระบุรหัสองค์กรในหน้าสมัครได้
- Admin เปิดเมนู **ผู้ใช้งานและสิทธิ์** เพื่ออนุมัติ
- Admin กำหนดบทบาทองค์กร, Event และจุด Start/CP/Finish
- ผู้ใช้เห็นเฉพาะเมนูและข้อมูลตามที่ได้รับมอบหมาย
- RLS และ RPC ตรวจสิทธิ์ซ้ำในฐานข้อมูล ไม่ใช่ซ่อนเมนูอย่างเดียว

### Dashboard แบบละเอียด
แสดงข้อมูลนักแข่ง:
- ชื่อ นามสกุล เลข BIB
- ประเภทและรุ่นการแข่งขัน
- เวลาเช็กอิน เวลาเริ่ม จุดล่าสุด เวลาเข้าเส้น
- ระยะเวลาที่ใช้
- อันดับเข้าเส้นรวม อันดับประเภท และอันดับรุ่น
- ประวัติการสแกนทุกจุด

### หน้าสาธารณะสำหรับนักวิ่ง
เปิด `results.html` แล้วเลือก Event และกรอกเลข BIB เพื่อดู:
- เวลาเช็กอิน / เริ่ม / เข้าเส้น
- เวลาที่ใช้
- อันดับรวม / ประเภท / รุ่น
- ประวัติผ่าน Start, CP และ Finish

### SQL เพิ่มเติม
รันไฟล์ `supabase/005_user_access_dashboard_public_results.sql` หลังไฟล์ 001-004

### ผู้ดูแลระบบคนแรก
หากเป็นฐานข้อมูลใหม่ บัญชีแรกจะเห็นปุ่ม **ตั้งเป็นผู้ดูแลระบบคนแรก** ในหน้า `pending.html` เพียงครั้งเดียว หลังจากนั้นทุกบัญชีต้องรอ Admin อนุมัติ

### การติดตั้งและอัปเกรด
อ่านไฟล์ `UPGRADE_TO_V5.txt` ก่อนอัปโหลด โดย V4 ที่มีฐานข้อมูลเดิมให้รันเฉพาะ SQL 005 หลังสำรองข้อมูลแล้ว
